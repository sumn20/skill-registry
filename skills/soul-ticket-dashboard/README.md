# Soul 工单深度分析工具

批量为 Soul 语音/视频反馈工单生成 TRTC 仪表盘链接、通话详情链接，并自动分析通话事件和音频指标，推断问题根因。

## 环境准备

```bash
# 安装依赖
pip install -r requirements.txt

# 安装 Playwright 浏览器（获取通话详情需要）
playwright install chromium
```

## 快速开始

### 1. 基础模式 — 仅生成仪表盘链接
```bash
python3 scripts/gen_dashboard.py <input.xlsx> <output.xlsx>
```

### 2. 详情模式 — 额外获取通话详情链接
```bash
python3 scripts/gen_dashboard.py <input.xlsx> <output.xlsx> --detail
```
> 首次运行需在弹出的浏览器中完成 OA/iOA 验证，之后自动复用登录态。

### 3. 深度分析（推荐）— 完整功能
```bash
python3 scripts/gen_dashboard.py <input.xlsx> <output.xlsx> --deep
```

`--deep` 一次完成所有功能：
- ✅ 仪表盘搜索页 URL（V列）
- ✅ 通话详情页 URL（W列）
- ✅ 排查结论：设备信息 + 异常事件 + 音频指标（T列）
- ✅ 独立根因推断（U列）

## 输入 Excel 要求

必需列（按列名自动定位）：
| 列名 | 说明 |
|------|------|
| 反馈渠道 | 语音匹配 / 私聊语音 / 视频匹配 |
| channelId | 通话房间标识 |
| 反馈时间 | 用户反馈时间 |

深度分析还需要：
| 列名 | 说明 |
|------|------|
| 反馈文本 | 包含 peerUIDs、问题类型等信息 |

## 输出列说明

| 列 | 内容 |
|----|------|
| T列 | 排查结论（问题类型 + 通话状态 + 设备信息 + 异常事件 + 音频指标） |
| U列 | 根因（独立列，方便筛选和统计） |
| V列 | TRTC 仪表盘搜索页 URL（可点击） |
| W列 | TRTC 通话详情页 URL（可点击） |

## 检测能力

深度分析可检测以下问题根因：
- APP 前后台切换导致音频中断
- 本地静音/停止采集
- 音频采样率异常变更
- 首帧音频缺失
- 在后台状态下退出通话
- 音频采集/播放音量异常（静音、断续）

## 性能参考

- 基础模式：秒级完成
- 深度分析：每条约 8-10 秒（5-6 个 API 调用），200 条约 25 分钟

## 故障排除

| 问题 | 解决方案 |
|------|---------|
| OA 登录超时 | 5分钟未验证会超时，重新运行即可 |
| API 无返回 | 房间号不存在或时间范围不匹配，自动跳过 |
| 锁文件残留 | 脚本会自动清理 `~/.soul-trtc-profile` 下的锁文件 |
